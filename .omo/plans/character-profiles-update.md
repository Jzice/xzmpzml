| 类型 | 缺失内容 | 影响 |
|------|----------|------|
| 角色 | 修为进阶时间线 | 无法追踪角色成长 |

| 势力 | 兴衰史 | 历史沿革不清 |
| 势力 | 资源领地 | 势力范围模糊 |

## Work Objectives

### 核心目标
1. 为50个核心角色建立**标准化关键信息档案**
2. 为20个势力建立**组织结构与兴衰史**
3. 统一**档案格式标准**

### 档案标准化要求

#### 角色档案 - 必须包含
| 字段 | 说明 | 优先级 |
|------|------|--------|
| 基本信息 | 姓名/性别/修为/本命/势力 | 必须 |
| 修为时间线 | 各阶段修为及达成时间 | 高 |
| 本命详情 | 本命名称/品阶/天赋/克应 | 高 |
| 首次出场 | 精确到章节 | 必须 |
| 死亡记录 | 精确章节或"存活" | 必须 |
| 势力归属 | 当前及历史归属 | 中 |
| 设定关联 | 相关设定条目 | 中 |
| 重要关系 | 师承/仇敌/配偶/子女 | 必须 |
| 关键事件 | 按时间顺序的大事记 | 高 |

#### 势力档案 - 必须包含
| 字段 | 说明 | 优先级 |
|------|------|--------|
| 基本信息 | 名称/类型/等级/所在 | 必须 |
| 组织架构 | 掌门/长老/弟子层级 | 必须 |
| 核心成员 | 历代掌门/现掌门/元老 | 必须 |
| 势力范围 | 领地/附属/盟友 | 高 |
| 兴衰史 | 建立/鼎盛/衰落/复兴 | 高 |
| 主要事件 | 按时间顺序的大事 | 必须 |
| 设定关联 | 相关设定 | 中 |

## Execution Strategy

### Wave Structure

```
Wave 1-5: 角色档案完善 (每波10人)
Wave 6-10: 角色档案完善 (续)
Wave 11-15: 势力档案深化
Wave 16-20: 关系网络建立
Wave 21-25: 索引与交叉引用
```

### 并行策略
- 每波分析2-3个相关角色（同一势力/同一事件）
- 势力档案可独立并行
- 最终整合阶段需要串行

1. **首次出现**: 精确到"第X卷第Y章"
2. **修为描述**: 文中直接描述或通过战斗/对比推断
3. **本命**: 文中提及的本命名称、天赋、克应
4. **关系建立**: 师徒、盟友、敌对关系的建立时间点
5. **关键事件**: 死亡、晋升、重大抉择

### 势力信息提取要点

1. **建立时间**: 文中提及的创立时间
2. **历代掌门/家主**: 按顺序记录
3. **组织结构**: 文中描述的内部架构
4. **势力范围**: 领地、附属、影响范围
5. **重大事件**: 战争、变革、衰落/复兴
