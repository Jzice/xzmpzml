# 人物档案核查验证计划

## 目的

确保更新的角色档案内容与小说原文完全一致，避免主观解读或错误信息。

---

## 一、核查流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  章节阅读   │ -> │  信息提取   │ -> │  档案更新   │ -> │  三级核查   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                     │
                                              ┌───────────────────────┘
                                              ▼
                                       ┌─────────────┐
                                       │  质量问题   │
                                       │  修复循环   │
                                       └─────────────┘
```

---

## 二、三级核查体系

### L1: 执行者自查 (Self-Check)

| 检查项 | 检查内容 | 标准 |
|--------|----------|------|
| 必填字段 | 姓名、性别、修为、势力 | 不能为空 |
| 格式规范 | Markdown表格格式 | 正确对齐 |
| 命名规范 | 文件名符合规则 | `{修为}/{编号}-{姓名}.md` |
| 首次出场 | 卷号章节号 | 格式: 第X卷第X章 |

**自查清单**:
- [ ] 所有必填字段已填写
- [ ] 修为等级正确（凡人/练气/筑基/金丹/元婴/化神）
- [ ] 势力名称与原文一致
- [ ] 文件命名符合规范

---

### L2: 交叉核查 (Cross-Verification)

**执行者**: 独立的核查agent

| 核查类型 | 核查内容 | 方法 |
|----------|----------|------|
| 原文核对 | 角色信息是否来自原文 | 搜索原文关键词 |
| 事实核对 | 修为、事件是否准确 | 对照章节内容 |
| 关系核对 | 师承、敌友关系 | 检查多处描述 |
| 时间线核对 | 进阶时间、章节对应 | 对照目录 |

**交叉核查表**:

| 角色 | 原文引用 | 提取信息 | 核对结果 | 差异说明 |
|------|----------|----------|----------|----------|
| 齐休 | "齐休练气二层..." | 练气二层 | ✓ | - |
| 齐掌门 | "齐掌门练气大圆满" | 练气大圆满 | ✓ | - |

**核对标准**:
- [ ] 每条信息必须有原文支撑
- [ ] 修为描述必须与原文一致
- [ ] 角色关系必须有至少2处原文佐证

---

### L3: 最终审核 (Final Review)

**执行者**: 主agent或用户

| 审核内容 | 审核要点 |
|----------|----------|
| 逻辑一致性 | 角色命运是否合理 |
| 完整性 | 是否有遗漏的重要角色 |
| 更新同步 | 相关的README是否同步更新 |

**最终审核清单**:
- [ ] 该卷所有主要角色已建立/更新
- [ ] 角色之间关系正确
- [ ] 势力归属正确
- [ ] 相关README索引已更新
- [ ] 无重复角色（相同人物不同档案）

---

## 三、问题修复循环

### 问题类型与处理

| 问题类型 | 示例 | 处理方式 |
|----------|------|----------|
| 信息缺失 | 缺少修为 | 标记"待查"，继续推进 |
| 信息错误 | 修为写错 | L2核查发现 → 修正 |
| 关系不清 | 师承不明 | 标注"推测"并注明 |
| 原文矛盾 | 不同章节描述冲突 | 记录矛盾处，标注"存疑" |

### 修复流程

```
发现问题 -> 定位原文 -> 确认正确信息 -> 修正档案 -> 再次核查
```

## 四、原文引用规范

### 引用格式

```
【原文】{第X卷第X章}："{角色对话/描述}"
【提取】：{提取的信息}
【状态】：✓正确 / ✗错误 / ⚠存疑
```

### 引用示例

```
【原文】第1卷第1章："这齐休虽只是双属性天根，但心智坚韧，倒是个好苗子。"
【提取】：齐休为双属性天根
【状态】：✓正确

【原文】第1卷第6章："齐掌门为门派操劳一生，最终因旧伤复发而亡。"
【提取】：齐掌门死亡原因：旧伤复发
【状态】：✓正确
```

## 六、质量指标

### 量化目标

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 信息完整率 | ≥95% | 必填字段/总字段 |
| 原文匹配率 | ≥98% | 核查通过数/总信息数 |
| 角色覆盖率 | ≥90% | 实际角色数/预估角色数 |

### 质量追溯

- 所有档案需保留原文引用记录
- 重大修订需记录修订原因
- 定期汇总常见错误类型

## 八、注意事项

1. **不添加主观解读**: 只记录原文事实，不推测心理活动
2. **多处验证**: 重要信息需在2处以上原文确认
3. **存疑标记**: 无法确认的信息标记"待查"
4. **及时记录**: 提取信息时立即记录原文出处
