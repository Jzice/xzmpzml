## 二、角色信息提取标准

### 必须提取的信息

| 信息类别 | 字段 | 示例 |
|----------|------|------|
| 基本信息 | 姓名、性别、修为 | 齐休、男、元婴 |
| 势力归属 | 门派/家族/散修 | 楚秦门、齐云派 |
| 首次出场 | 卷号、章节 | 第1卷第1章 |
| 角色关系 | 师承、亲属、敌友 | 师父：齐掌门 |
| 主要事迹 | 关键事件 | 南迁、黑河坊建立 |

### 修为等级体系

- 凡人 → 练气 → 筑基 → 金丹 → 元婴 → 化神
- 每个境界有前期、中期、后期、大圆满

## 三、核查验证计划表

### 三级核查机制

| 核查级别 | 执行者 | 核查内容 | 通过标准 |
|----------|--------|----------|----------|
| L1 自查 | 执行agent | 提取信息完整性 | 必填字段齐全 |
| L2 交叉 | 另一agent | 原文事实核对 | 逐句对应 |
| L3 最终 | 用户/主agent | 逻辑一致性 | 角色关系正确 |

### 核查要点

1. **事实核查**: 角色名、修为、势力是否与原文一致
2. **关系核查**: 师承、亲属、敌友关系是否准确
3. **时间线核查**: 修为进阶、事件发生时间是否正确
4. **逻辑核查**: 角色命运走向是否合理

### 验证引用格式

```
【原文】: "xxx" (第X卷第X章)
【提取】: xxx
【核对】: ✓/✗
【备注】: 
```

## 六、执行策略

### 并行处理

由于各章节相对独立，可以：
1. 多个agent同时读取不同章节
2. 提取信息后汇总
3. 批量更新角色档案

### 优先级

1. **主线角色**: 齐休及相关核心人物
2. **重要配角**: 各门派掌门、长老
3. **次要角色**: 散修、小门派

### 质量控制

- 每完成5章进行一次小审核
- 每完成1卷进行一次大审核
- 定期与用户同步进度
